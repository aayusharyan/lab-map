# Claude Code Instructions

Rules and constraints for AI agents working on this codebase.

## Hard Rules

1. **Keywords sync** - GitHub topics and `package.json` keywords must always match (max 20)
2. **No homelab terminology** - Target audience is both homelab users AND enterprise; use neutral language
3. **Not an npm package** - This is an end-user website/visualizer, not a library for others to install
4. **pnpm only** - npm/yarn are blocked; always use `pnpm` commands
5. **Image-only nodes** - All graph nodes use `shape: 'image'`; never use shape-based nodes (box, ellipse, dot, etc.)

## Naming Conventions

- **Booleans**: Use `is...` prefix (`isOpen`, `isLoading`, `isVisible`)
- **Throwing functions**: Include `OrThrow` suffix when function throws or propagates throws (`fetchJsonOrThrow`, `getNodeTypeOrThrow`)
- **Comments**: Use `/* */` syntax in TS/JS/CSS, never `//` for documentation

## Directory Quick Reference

```
config/             ← User configs (committed, Docker ships these)
public/data/        ← Runtime data (gitignored, generated by watcher)
schema/             ← JSON Schema validation
public/node-icons/  ← Valid node icons (source of truth)
```

## Before You Code

1. Run `pnpm install` if dependencies changed
2. Run `pnpm dev` to start dev server + watcher
3. Check `schema/` for data file contracts
4. Check `public/node-icons/` for valid node types
